# RELY: Reliability of Evaluation for small-sample radiomics

**Sampling Variance, Not Dimensionality: A Pre-Modeling Reliability Gate for Small-Sample Radiomics Evaluation.**

RELY is an open toolkit and benchmark for the question every small-sample radiomics study should ask *before* training a model: **will the resulting AUC be believable?** It shows that the optimism in reported radiomics performance is a **sampling-variance** phenomenon, a closed-form function of the class counts, and is essentially **unrelated to feature dimensionality**. So the field's reflexive lever (reduce features) is the wrong one for evaluation reliability; only more data, especially more minority-class events, moves the quantity that matters.

> **Double-blind:** this repository contains **no author or institution identifiers**. Prior related write-ups are cited in the third person. (De-anonymize on publication.)

> **Public release:** this is a curated **code + benchmark** release. The single-center patient feature table for the AIS case study is **not distributed** (privacy/provenance); its outputs ship only as cached figures and aggregate `results/`. The load-bearing evidence, the 50-cohort benchmark, reproduces fully from public data via `radMLBench`.

## What RELY gives you
- **A reliability calculator** (`src/reliability_predictor.py`): from class counts (and a selection budget), the expected reported optimism, or the minority events needed to bound it; validated across 50 real cohorts.
- **A pre-modeling evaluability gate** (`src/screening_gate.py`): from class counts alone, before any modeling, flags cohorts whose honest evaluation will be indistinguishable from chance. It is a high-specificity warning (ROC-AUC **0.83**; out-of-sample operating point spec 0.86 / sens 0.56), so it misses about 44% of un-evaluable cohorts and a non-flag is not a clearance.
- **A high-variance-learner robustness check** (`src/run_radmlbench_rf.py`, `src/analyze_rf_robustness.py`): re-runs the whole sweep with random forests and no univariate preselection, confirming the dimensionality-irrelevance is not specific to the regularized linear pipeline.
- **An honest-evaluation audit** (`src/honest_eval.py`): leakage-free repeated stratified nested CV, Nadeau-Bengio corrected variance, permutation null, the optimism ladder.
- **A 50-cohort benchmark harness** (`src/run_radmlbench.py`) over radMLBench.
- **A pre-registration template** (`PREREGISTRATION.md`): freeze the primary model, metric, feature selection, and analysis *before* results; aligned with PROBAST and TRIPOD+AI.

## Key findings (all regenerated by this code)
- **Optimism is predicted by sampling variance, not dimensionality.** Across **50** real radiomics cohorts, the cherry-pick optimism is predicted by the closed-form Hanley-McNeil AUC standard error, **R²=0.73** (0.65 at a fixed null working point that shares nothing with the honest estimate; LOSO [0.71,0.79]; cluster-bootstrap [0.57,0.86]), far above **feature dimensionality (R²=0.09)**.
- **Robust to a high-variance learner.** Re-running the sweep with random forests and no univariate preselection (dimensionality free to drive overfitting) reproduces the structure: optimism still tracks the class-count SE (**R²=0.70**; 0.55 shared-term-free) while dimensionality stays irrelevant (**R²=0.08**, incremental **0.001** over the class counts); the two learners' per-cohort optimism correlates **r=0.62**.
- **Within-cohort controlled identification.** Holding the feature space fixed, the optimism ceiling rises whenever *either* class count shrinks (fix-n vary-events r=-0.60; fix-events vary-n r=-0.55; honest flat), so the driver is joint sampling variance, not minority events alone and not dimensionality. (This refuted an earlier "event-count" framing.)
- **Pre-modeling gate works.** ROC-AUC 0.83 (LOSO [0.82,0.85]) predicting "honest CI covers chance" from class counts alone, as a high-specificity warning.
- **Reliability calculator.** Bounding expected optimism at 0.10 needs about 214 minority events (prevalence 0.3); at 0.05, about 847, thresholds most studies never approach.
- **AIS worked example.** A real single-center acute ischemic stroke (AIS) cohort (n=183, 57 events): honest nested-CV AUC **0.47** (permutation **p=0.54**), yet a cherry-picked split reaches **0.67**; the 1004-feature model is no better than a 5-feature age+volume baseline (a paired Nadeau-Bengio-corrected test even ranks it modestly worse, p=0.035, but the marginal intervals overlap and we do not lean on a single small cohort). It is the toolkit's end-to-end worked example, not the load-bearing evidence (that is the 50-cohort benchmark).

## Reproduce
```bash
pip install -r requirements.txt           # Python 3.12; radMLBench; PyRadiomics not needed
python src/run_radmlbench.py --all         # 50-cohort optimism sweep -> results/radmlbench_sweep.csv
python src/reliability_predictor.py        # the predictor + calculator (Fig: reliability_validation)
python src/screening_gate.py               # the pre-modeling gate (Fig: screening_gate_roc)
python src/r_e.py --cohorts 4 --K 200 --reps 3   # within-cohort identification (Fig: C1_identify)
python src/run_radmlbench_rf.py            # random-forest robustness sweep -> results/radmlbench_sweep_rf.csv
python src/analyze_rf_robustness.py        # RF vs linear robustness summary -> results/rf_robustness.json
```
All randomness is seeded (master seed 20260625).

The AIS worked-example scripts (`honest_eval.py`, `ais_forensic.py`, `baselines.py`, `tables.py`) require the single-center patient feature table, which is **not distributed** in this release; their outputs ship as the cached `figures/AIS_forensic.*` and aggregate `results/*.json`.

## Data & scope
- The **AIS cohort** feature table is **not included** (single-center, patient-derived; original images/masks/extraction code and clinical records are lost). Only its aggregate outputs appear here (`results/`, `figures/`).
- External cohorts via **radMLBench** (`pip install radMLBench`, public, real radiomics), the load-bearing 50-cohort evidence.
